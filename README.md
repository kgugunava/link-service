# Link Service
Сервис для укорачивания ссылок, реализованный в рамках тестового задания в Ozon Tech. 

## Реализованный функционал
- Создание короткой ссылки по оригинальному URL
- Получение оригинального URL по короткому коду
- Идемпотентность: повторный запрос на сокращение того же URL возвращает тот же код
- Детерминированная генерация кодов: один и тот же URL всегда даёт один и тот же код
- Гарантия формата кода: ровно 10 символов из [A-Za-z0-9_], обязательно наличие хотя бы одной заглавной буквы, строчной буквы, цифры и символа подчёркивания
- Автоматическая обработка коллизий коротких кодов с fallback-логикой
- Настраиваемый TTL для ссылок с автоматической очисткой просроченных записей
- Два типа хранилищ: in-memory и PostgreSQL
- Graceful shutdown с корректным завершением всех фоновых процессов
- Структурированное логирование через log/slog
- Полное покрытие unit-тестами

### Конфигурации приложения
Конфигурация задается через .env файл. Пример можно увидеть в файле .env.example
```
# Тип хранилища: memory | postgres
# По умолчанию: memory
# При значении "postgres" необходимо корректно задать PG_DSN
STORAGE_TYPE=memory

# PostgreSQL DSN для подключения к базе данных
# Используется только при STORAGE_TYPE=postgres
# Формат: postgres://user:pass@host:port/db?sslmode=disable
# По умолчанию: postgres://user:pass@localhost:5432/shortener?sslmode=disable
PG_DSN=postgres://user:pass@localhost:5432/shortener?sslmode=disable

# Порт HTTP-сервера
# По умолчанию: :8080
PORT=:8080

# Время жизни короткой ссылки (TTL)
# Поддерживает форматы: "720h" (часы), "30m" (минуты), "30d" (дни), "1h30m"
# По умолчанию: 720h (30 дней)
URL_TTL=720h

# Интервал фоновой очистки просроченных записей
# Поддерживает форматы: "1h", "30m", "1h30m"
# По умолчанию: 1h (каждый час)
CLEANER_INTERVAL=1h

# Уровень логирования
# DEBUG=1 включает debug-логи (подробная информация о каждом запросе)
# DEBUG=0 или не задано — только info, warn, error
# По умолчанию: 0
DEBUG=0

# Формат логов: text | json
# text — человекочитаемый формат для локальной разработки
# json — структурированный JSON 
# По умолчанию: text
LOG_FORMAT=text
```
### Запуск приложения
Приложения запускается с помощью команды


## API

### POST /api/shorten
**POST Request**
```
POST /api/shorten
Content-Type: application/json
{
  "url": "https://example.com/path/to/resource?param=value"
}
```
**Responses**

```
201 Created
{
  "shorten_url": "Ab3_xK9mLp"
}
```
```
400 Bad Request - невалидный JSON или переданный URL
```
```
500 Internal Server Error - внутренняя ошибка сервера
```

### GET /api/original/:short_url

**GET Request**

```
GET /api/original/:short_url
```
**Responses**
```
200 OK
{
  "original_url": "https://example.com/path/to/resource?param=value"
}
```
```
400 Bad Request - невалидный переданная короткая ссылка
```
```
404 Not Found - короткая ссылка не найдена
```
```
500 Internal Server Error - внутренняя ошибка сервера
```

## Генерация коротких ссылок
Реализация в ```./internal/utils/generator.go```
- Детерминированный результат
- Каждая сгенерированная ссылка удовлетворяет требованиям из ТЗ
- Реализована обработка возможных коллизий

## TTL и очистка данных
Каждая ссылка имеет время жизни (TTL), настраиваемое в конфигурации приложения

## Тестирование
### Юнит-тестирование
Реализованная бизнес-логика покрыта юнит-тестами
Для запуска юнит-тестов с проверкой на гонку данных используется команда
```
go test -race -cover ./...
```
### Нагрузочное тестирование
Для нагрузочного тестирования приложения используется Grafana k6
Для проведения нагрузочного тестирования:
- в окне терминала запустить приложение командой ```docker compose up```
- в другом окне терминала запустить скрипт сценария нагрузочного тестирования командой ```k6 run load_test.js```
